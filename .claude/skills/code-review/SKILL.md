$ARGUMENTS のコードレビューを実施してください。

## レビュー観点

### 1. 正確性
- ロジックのバグや意図しない動作がないか
- エッジケース（空配列、null、undefined、0円）の処理漏れ
- 非同期処理のエラーハンドリングが適切か

### 2. 可読性・保守性
- 変数名・関数名が意図を明確に表しているか
- 関数が単一責任を持っているか
- 複雑なロジックに適切なコメントがあるか

### 3. パフォーマンス
- 不要な再レンダリングがないか（useCallback / useMemo の過不足）
- 不要なループやネストがないか

### 4. TypeScript 型安全性
- `any` の使用を避けているか
- `import type` を使っているか（verbatimModuleSyntax 必須）
- 型アサーションの乱用がないか

### 5. ワリタロ会計部 固有の観点
- リポジトリメソッドは `async/await` で統一されているか
- ID 生成に `generateId()`、日時に `nowISO()` を使っているか
- フォーム input は `text-base`（16px）になっているか（iOS ズーム防止）
- `window.prompt` / `window.confirm` を使っていないか（Modal コンポーネントを使う）
- `settlementService` の計算ロジックが `isSplit: true` の項目のみを対象にしているか

## 出力フォーマット

各指摘事項を以下の形式で報告してください：

```
[重要度: 🔴 重大 / 🟡 改善推奨 / 🔵 提案]
ファイル名:行番号
問題: ...
理由: ...
修正案: ...
```

最後に「総評」として全体的な品質評価を1段落でまとめてください。
